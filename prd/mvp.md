# 像素风5v5数值卡牌篮球游戏 MVP需求文档
## 1 项目基础
1.1 技术栈
后端：Node.js + Express + SQLite单文件数据库
前端：HTML5 + 原生ES6 JS + CSS像素样式
适配优先级：移动端优先，兼容PC桌面浏览器
美术风格：全局像素风（按钮、卡牌、头像、图标、面板）
运行模式：网页端账号密码登录，多玩家数据隔离存储

1.2 核心玩法
抽卡养成球员卡组 → 固定5位置编队 → AI三种模式对战 / 异步真人PVP数值结算对战
无实时动画、无实时联机、无跑动渲染，纯数值计算赛果+球员数据统计

## 2 账号系统
2.1 登录注册
登录凭证：唯一自定义游戏ID + 密码
玩家可自由设置展示昵称；底层数据库匹配、ID识别使用game_id，所有界面展示统一显示nickname
登录页使用背景图片(static/assets/backgroud.jpg)，登录/注册失败给出toast错误反馈
2.2 新手福利
新账号创建直接发放50抽次数；抽卡支持【单抽】、【十连抽】两种操作
2.3 玩家基础资源
金币、钻石、球星碎片（原升星碎片）；持久化存入玩家数据表
2.4 退出登录
主页提供退出账号功能，清除本地缓存跳转登录页

## 3 球员卡牌体系
### 3.1 品级、星级上限、升星规则
品级分为 B / A / S / SS / SSS
星级上限：统一5星
升星合成条件：**同品级、同角色、同星级**2张卡牌合成，星级+1；跨品级/不同角色无法合成
成长增量：
- B级：每升1星，全属性+4
- A级：每升1星，全属性+6
- S级：每升1星，全属性+8
- SS级（橙色）：每升1星，全属性+10，基础属性=S级×1.1
- SSS级（红色）：每升1星，全属性+12，基础属性=S级×1.3
最终属性值 = 角色品级基础数值 + 星级成长数值

### 3.2 S/SS/SSS角色天赋简化规则
S、SS、SSS角色均享有单一主属性结算阶段+20%倍率加成；B/A无天赋加成
SS级角色：大学时期·陶二(SF)、单身·小亮(SF)、黄毛·阿晟(PF)
SSS级角色：极致·华仔(C)
摇摆双位置机制：陶二(SG/SF)、阿晟(PF/C)可放入两个对应位置任意槽位

### 3.3 B/A角色精简配置
每个位置(PG/SG/SF/PF/C)固定配置：3个A级角色、5个B级角色
内置固定名称、固定位置、固定基础属性，无动态随机生成逻辑

### 3.4 卡牌界面
卡牌仓库页(cards.html)：
- 纯展示功能，按位置/品级筛选，分页浏览，点击查看详情弹窗
- 查看内容：品级标签、位置、星级、8项属性条状图(满分100)、S角色天赋说明、星级成长预览表
- S级球员显示专属图片(static/assets/目录下以拼音命名)
- 球星碎片兑换功能：50碎片兑换B级，200碎片兑换A级，1000碎片兑换S级

### 3.5 阵容站位与升星
阵容页(lineup.html)集成以下功能：
1. **阵容编排**：仅允许保存**一套出战阵容**，5个标准位置（PG/SG/SF/PF/C）
2. **摇摆位**：摇摆球员可放入两个对应位置任意槽位
3. **阵容战力**：5槽填满时实时显示阵容总战力（每张卡牌战力=全属性平均值×10，阵容战力=五张汇总）
3. **持有卡牌**：展示玩家全部卡牌，支持位置/品级筛选
4. **卡牌详情**：点击持有卡牌弹出详情弹窗（属性条状图+S天赋说明）
5. **一键合成**：自动将所有可合成卡牌升到最高星，循环配对合成
6. **一键分解**：批量选择卡牌分解，支持按品级筛选
   - B级分解：50金币 + 50碎片
   - A级分解：100金币 + 100碎片
   - S级分解：100钻石

## 4 抽卡经济与卡池概率
4.1 卡池划分
1）普通金币池（消耗金币抽取）
概率：B 67%、A 30%、S 3%
2）高级钻石池（消耗钻石抽取）
B/A/S混合，S出货概率高于金币池（配置高于10%）
4.2 十连规则
十连等价消耗10次单抽资源，无额外复杂保底机制
4.3 安慰阶梯保底
连续多抽未产出S卡时，下一抽小幅上调S概率，阶梯固定数值微调
4.4 抽卡附属功能
- 展示最近10抽记录列表
- 玩家主表存储全局总抽卡数、S卡总出货数量，用于数值统计平衡
- S级抽卡闪光动画（炉石传说传说卡风格）

## 5 三种AI电脑对战模式（固定三类）
1）日常训练
对手：根据我方阵容实力动态随机生成B/A混合AI队
次数限制：每日30次
奖励：金币50~200随机（概率偏低于150）+ 少量碎片
2）梯队联赛
对手：青铜→白银→黄金梯度预设固定AI阵容，每日有挑战次数上限
奖励（胜利掉落，随等级增加）：
- 青铜：碎片20 + 钻石20
- 白银：碎片50 + 钻石50
- 黄金：碎片100 + 钻石100
3）精英挑战赛
对手：预设高星S顶配固定AI强队，每日仅1~2次挑战机会
奖励（统一）：金币500 + 碎片200 + 钻石200
AI优化：仅联赛、精英存储预设强队；日常训练实时随机生成，不入库冗余AI队伍

## 6 真人异步PVP规则
6.1 对战逻辑
读取其他玩家已保存的唯一出战阵容作为对手快照，双方无需同时在线，数值结算胜负
战力优势修正：攻防综合属性差异影响命中率（每1点差异≈1.5%命中率修正，上限±6%）
6.2 对手选择
随机从数据库抓取拥有完整阵容的真实玩家作为可挑战列表，支持"换一批"刷新
挑战面板显示我方阵容及总战力、对手阵容及战力
6.3 积分规则
仅战胜真人对手增加赛季积分；被他人挑战落败，自身积分不扣除
6.4 对战冷却
真人PVP每次对战结束强制冷却5秒，才可再次发起挑战
6.5 每日次数限制
PVP每日限10次对战
6.6 数据存储限制
1. 全局match_records表保存所有AI/PVP对局：胜负、双方总分、积分变动、对局时间、对局类型
2. 单场JSON存储6名球员（己方3+敌方3）详细赛场数据：得分、篮板、助攻、抢断、盖帽
3. 对战日志（原防守日志）展示PVP进攻+防守比赛记录，保留最近20条
6.7 赛季体系
固定周期赛季，赛季结束积分重置；依据赛季积分档位发放钻石、碎片奖励
6.8 排行榜
- 赛季积分TOP20排行榜
- 每日单项数据榜：得分、篮板、助攻、抢断、盖帽，按球员维度统计
6.9 每日MVP
每天20:00自动结算，当日PVP获胜次数最多的玩家获得MVP称号
奖励：300钻石
PVP页面展示当日MVP获得者信息

## 7 配套功能
1. 成就系统：固定4项成就清单，完成后可领取钻石/抽卡次数奖励，已领取标记防重复
   - 首十连：完成首次十连抽卡 → 50钻石
   - S三星：将任意S卡升至3星 → 10次抽卡
   - 首胜真人：PVP首次战胜真人 → 30钻石
   - 黄金联赛通关：黄金联赛取得胜利 → 100钻石
2. 卡牌详情弹窗：点击卡牌展示完整8项属性条状图、星级成长预览、S角色天赋说明
3. 抽卡记录：展示近10抽卡牌产出
4. 全局抽卡统计：总抽次数、总S出货数量存档

## 9 SQLite数据表结构
### 9.1 players 玩家主表
player_id(自增主键)
game_id(登录唯一ID)
password
nickname
gold 默认1000
diamond 默认0
upgrade_shard 默认0（球星碎片）
free_draws 默认50(新手福利抽卡次数)
total_draw 默认0 总抽卡次数
total_s 默认0 累计抽到S卡数量
current_season 默认1 当前赛季编号
pity_count 默认0 安慰阶梯计数
last_pvp_time 默认0 上次PVP时间戳

### 9.2 player_cards 玩家持有卡牌
card_uid(唯一主键)
player_id(关联玩家)
pos 位置 PG/SG/SF/PF/C
grade B/A/S
role_name 角色名称
star 当前星级 默认1

### 9.3 team_deck 唯一出战阵容
deck_id
player_id(唯一)
slot1_card 槽1(PG)卡牌uid
slot2_card 槽2(SG)卡牌uid
slot3_card 槽3(SF)卡牌uid
slot4_card 槽4(PF)卡牌uid
slot5_card 槽5(C)卡牌uid

### 9.4 match_records 全局对战记录
match_id 对局主键
attacker_id 发起对战玩家id
defender_id 被挑战玩家id，AI对局留空
match_type 标识：train/league/elite/pvp
win_id 胜利方id
att_score 进攻方总分
def_score 防守方总分
player_stats_json 六位球员比赛数据JSON字符串
point_change 本场积分变动数值
create_time 对局时间戳

### 9.5 player_season 赛季数据
player_id(唯一)
season_points 当前赛季积分
defense_log JSON存储近20条防守对局快照
season_num 所属赛季

### 9.6 player_achieve 成就表
player_id
achievement_key 成就唯一标识
is_finish 是否完成 默认0
reward_get 是否已领取奖励 默认0
主键：(player_id, achievement_key)复合主键防重复

### 9.7 league_ai_teams 联赛AI队伍
team_id(自增主键)
tier 梯度标识：bronze/silver/gold/elite_1/elite_2
team_name 队伍名称
roster_json 阵容JSON

### 9.8 league_daily_challenges 联赛每日挑战记录
player_id
tier 梯度标识
challenge_count 当日已挑战次数
last_date 最后挑战日期

### 9.9 daily_mvp 每日MVP记录
id(自增主键)
mvp_date 日期（唯一）
player_id MVP玩家id
nickname MVP玩家昵称
win_count 当日胜利次数
settle_time 结算时间戳

## 10 战斗结算流程
1. 读取双方5张卡牌完整属性，叠加星级、品级数值、S/SS/SSS角色天赋20%主属性加成
2. 计算双方攻防综合评分，产生战力优势修正（命中率±6%上限）
3. 随机模拟70回合攻防出手逻辑，计算双方总得分判定胜负
4. 批量生成10名球员单场详细数据面板
5. 根据对局类型发放对应资源；仅PVP胜利增加赛季积分+钻石

## 11 移动端强制适配规范
1. 全局rem弹性自适应布局，禁止固定px宽高
2. 按钮、卡牌触控尺寸放大，完全兼容触屏，不依赖hover样式
3. 像素素材多倍率缩放，保证无模糊
4. 登录、改名输入框适配手机软键盘，无页面遮挡
5. 小屏自动折叠次要菜单，核心对战/抽卡/阵容面板优先展示

## 12 明确MVP不做范围（杜绝超额开发、减少Token消耗）
1. 无实时联机、无聊天社交
2. 无充值、商城、皮肤外观
3. 无多套阵容存档切换
4. 无Canvas球员跑动动画，仅数值面板展示赛果
5. 无全队羁绊复杂加成

# 全局项目开发标准
## 1 技术强制约束
1. 前端：纯HTML5 + ES6原生JS + CSS，禁止引入React/Vue/Three.js等第三方框架
2. 后端：Node Express + SQLite单文件数据库，禁止切换MySQL/PostgreSQL
3. 美术统一像素风，所有界面元素像素绘制风格，禁止写实/二次元平滑画风
4. 移动端优先适配，布局使用rem弹性方案，兼容PC端

## 2 代码编写规范
1. 语义化变量、常量，所有固定数值抽离配置常量，禁止魔法数字散落代码
2. 代码分层隔离：前端分页面模块、工具模块；后端分路由、控制器、数据库操作
3. 注释仅保留核心公式、判定规则、边界逻辑，删除冗余无效注释
4. 复用通用工具函数，禁止复制粘贴重复逻辑代码
5. 单次指令仅最小范围修改目标文件，禁止无差别全局重构、自动优化无关代码

## 3 账号展示硬性规则
1. 底层逻辑、数据库匹配使用game_id唯一识别玩家
2. 所有游戏界面：对战记录、排行榜、对战日志、阵容展示全部渲染nickname昵称
3. 新玩家固定开局50抽，抽卡只有单抽、十连两种交互

## 4 阵容站位强制规则
1. 玩家仅拥有一套出战卡组，无多套存储
2. 5个标准位置：槽1=PG、槽2=SG、槽3=SF、槽4=PF、槽5=C
3. 摇摆双位置球员可放入两个对应位置任意槽位
4. 五槽填满合规卡牌才可触发对战

## 5 Vibe Coding协作规则
1. 所有开发需求严格对齐prd/mvp.md，禁止擅自拓展MVP以外功能
2. 迭代只做需求内最小有效改动，不自动新增特性
3. 报错优先修复核心功能，不并行拓展额外玩法

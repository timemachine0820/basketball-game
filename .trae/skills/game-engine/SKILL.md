# 前端通用引擎&适配规范
## 1 移动端布局标准
1. 根标签设置rem基准尺寸，全部宽高、间距使用rem单位，禁止大量固定px
2. 触摸按钮最小点击尺寸适配手机触控，移除所有hover专属样式，改用touch点击事件
3. 小屏自动折叠侧边次要菜单，核心功能面板置顶优先渲染
4. 输入框适配软键盘，登录、改名输入区域不会被键盘遮挡

## 2 页面渲染规范
1. 像素图片统一缩放处理，保证多设备清晰度无模糊
2. 弹窗、卡牌、列表组件封装复用，统一样式结构
3. 所有接口请求封装统一请求工具，携带玩家登录身份校验
4. S级球员详情页显示专属图片(static/assets/目录下拼音命名)
5. S级抽卡闪光动画：s-glow-pulse光晕脉冲 + s-border-flash边框闪烁 + s-text-glow文字发光

## 3 本地缓存与交互
1. 登录状态、玩家基础信息做前端轻量化缓存，每次打开先校验登录态
2. 抽卡记录、卡牌列表、阵容数据从后端拉取，不长期本地持久覆盖数据库
3. 点击交互增加防抖，防止短时间重复点击触发多次请求

## 4 通用工具能力模板
1. 数字格式化、时间戳转展示时间
2. 弹窗提示、成功/失败反馈统一样式（toast）
3. 列表渲染循环模板，卡牌卡片渲染模板
4. 排行榜、对战记录列表渲染通用结构

## 5 PVP页面Tab结构
1. 挑战对手：我方阵容+对手列表+挑战按钮
2. 排行榜：赛季积分TOP20
3. 今日数据：5个子标签（得分/篮板/助攻/抢断/盖帽）切换
4. 对战日志：进攻+防守记录，可展开查看球员数据

## 6 阵容页面功能区
1. 阵容编排区：3槽位卡牌拖放/选择
2. 战力显示区：实时计算显示阵容总战力
3. 卡牌操作区：一键合成 + 一键分解按钮
4. 持有卡牌区：筛选+卡牌网格，分解模式下可点选
